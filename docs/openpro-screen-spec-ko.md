# OpenPro 화면별 상세 기획서

## 1. 문서 목적

이 문서는 OpenPro의 화면 단위 동작을 기획서 수준으로 정리한 문서다. 터미널 UI와 대화형 다이얼로그를 실제 화면으로 간주하고, 각 화면의 진입 조건, 구성 요소, 검증 규칙, 상태 변화, 예외 처리를 정리한다.

## 3. 공통 화면 규칙

### 3.1 공통 입력 규칙

- Enter: 선택 또는 확정
- Esc: 취소 또는 이전 단계 이동
- 화살표: 리스트 이동
- 일부 화면은 `x`, `f`, `y/n` 같은 전용 단축키를 가진다.

### 3.2 공통 상태 표현

| 상태 | 의미 |
|---|---|
| loading | 데이터 조회 중 |
| empty | 표시할 데이터 없음 |
| updating | 저장 또는 변경 적용 중 |
| error | 오류 발생 |
| success | 완료 직후 안내 상태 |

### 3.3 공통 예외 처리 원칙

- 빈 데이터는 예외가 아닌 정상 상태로 처리한다.
- 외부 연동 실패는 가능한 한 사용자가 바로 조치할 수 있는 메시지를 보여준다.
- 설정 저장 실패는 현재 상태를 유지하고 오류 메시지를 노출한다.

## 4. 화면 상세

### SCR-001 메인 대화 화면

| 항목 | 내용 |
|---|---|
| 화면명 | 메인 대화 화면 |
| 진입 경로 | `openpro`, `bun run dev` |
| 목적 | 사용자 프롬프트 입력과 도구 실행, 응답 스트리밍을 제공 |
| 사용자 유형 | 일반 사용자, 개발자, 운영자 |
| 사전 조건 | CLI 기동 및 provider 검증 완료 |

#### 화면 구성 요소

| 요소 | 설명 |
|---|---|
| 입력 영역 | 자연어 프롬프트 또는 슬래시 명령 입력 |
| 응답 영역 | 모델 답변과 도구 실행 결과 스트리밍 |
| 상태/보조 표시 | 진행 메시지, 권한 질의, task 알림 등 |

#### 주요 동작

1. 사용자가 프롬프트를 입력한다.
2. 시스템이 필요한 도구와 권한 여부를 판단한다.
3. 응답과 도구 결과를 순차적으로 표시한다.
4. 세션 로그에 즉시 저장한다.

#### 유효성 검증 및 예외

| 항목 | 처리 |
|---|---|
| 빈 입력 | 무시 또는 명령 도움말 유도 |
| provider 오류 | 시작 단계에서 차단되어 화면 진입 불가 |
| 도구 권한 필요 | 권한 승인 화면으로 전환 |

#### 상태 변화

- 대기 → 입력 중 → 처리 중 → 응답 완료
- 처리 중 → 권한 대기
- 처리 중 → 오류 출력

#### 테스트 포인트

- 일반 프롬프트 응답 생성
- 슬래시 명령 라우팅
- 장문 대화 후 로그 저장
- 도구 실행 결과 병합 표시

## 6. 화면 설계 시 유의사항

- 터미널 UI 특성상 상태 전환이 빠르므로, loading/error/success 메시지가 짧아도 의미가 분명해야 한다.
- 같은 명령이라도 인증 상태, 환경 상태, 데이터 개수에 따라 완전히 다른 화면 분기가 발생할 수 있다.
- Esc, Enter, 화살표, 문자 키 기반 조작은 문서와 도움말에서 일관되게 안내해야 한다.
- 설정 화면처럼 중첩 메뉴가 있는 경우 Esc 소유권 충돌을 방지해야 한다.
